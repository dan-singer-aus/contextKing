# Contributing

## What this project is

contextKing is a **system** — a set of conventions, file structures, and workflows for managing Claude context across sessions. It is not a content library. The preferences, style rules, and stack files are yours to fill in; the value is in the structure itself.

## What kinds of contributions are welcome

- **Structural improvements** — better ways to organise context, new file types, improved conventions
- **New workflow patterns** — e.g. new thread evolution types, better project context templates
- **Bug reports** — things that don't work as described, broken conventions, outdated instructions
- **Documentation improvements** — clearer explanations, better examples

## What won't be merged

- Personal preference content (your specific naming conventions, your stack choices, etc.)
- Changes that make the system more opinionated about things that are genuinely personal

## Opening a PR

- Describe what changed and why
- Note which version of Claude Code you tested with
- Keep changes focused — one structural concern per PR

## A note on response times

This is a personal project maintained in my spare time. Response to issues and PRs may be slow — I'll get to them when I can.
